## iOS Dev Weekly — Issue #1

### Opening

Async/await is no longer an experiment — teams are actively trading terse call sites for different runtime semantics. If you migrate networking without thinking about cancellation, testing, and observability, you will introduce subtle bugs that only show up under load or in CI.

---

### This Week's Big Story

**Migrate URLSession to Swift async/await**

Swapping URLSession completion handlers for async/await cleans up call sites fast but silently changes cancellation semantics and how errors bubble. That shift forces engineering choices around testability, rollout strategy, and metric capture. The piece lays out a pragmatic, low-risk path that keeps production correctness while letting you adopt Swift concurrency incrementally.

### Swift Snippet of the Week

```swift
import SwiftUI
import Foundation
import Observation

// Simple Decodable model matching a common public test API
struct Post: Identifiable, Decodable {
    let userId: Int
    let id: Int
    let title: String
    let body: String
}

// Centralized error type for networking layer
enum NetworkError: Error, LocalizedError {
    case invalidURL
    case invalidResponse(statusCode: Int)
    case decodingError(Error)
    case other(Error)
    
    var errorDescription: String? {
        switch self {
        case .invalidURL: return "Invalid URL."
        case .invalidResponse(let code): return "Server returned status code \(code)."
        case .decodingError(let e): return "Decoding failed: \(e.localizedDescription)"
        case .other(let e): return e.localizedDescription
// ... (truncated for newsletter)
```

This pattern matters because centralizing error shapes and models makes migration decisions (cancellation, retries, observability hooks) local and repeatable across your codebase.
