## iOS Dev Weekly — Issue #7

### Opening

Ad-hoc Tasks and callback-heavy networking still show up in new codebases, and they quietly cause crashes, wasted battery, and impossible-to-repro UI bugs. Making task lifetimes explicit and owning cancellable scopes is the sort of pragmatic hygiene that pays off immediately in stability and testability.

---

### This Week's Big Story

**Structured Concurrency Patterns for SwiftUI Tasks**

Conversion from floating Tasks and completion handlers to structured concurrency is about more than syntax; it's about making lifecycle boundaries explicit so background work dies where you expect. When you own the task scope you avoid UI updates after navigation and you make cancellations testable. The patterns below target incremental, low-risk refactors that stop the common runtime footguns.

@Observable class LegacyVM {
    var text: String = ""
    func fetch() async {
        let (data, _) = try! await URLSession.shared.data(from: URL(string: "https://example.com")!)
        text = String(decoding: data, as: UTF8.self)
    }
}
struct LegacyView: View {
    @State private var vm = LegacyVM()
    var body: some View {
        Text(vm.text)
