## iOS Dev Weekly — Issue #1

### Opening

Most mature apps carry navigation debt: push/pop calls, segues, and small hacks that made a feature ship instead of make the system resilient. Teams are quietly wrestling with deep links, state restoration, and modal coordination while debating whether SwiftUI's NavigationStack is a fit or just another refactor risk. This issue walks a practical path for migrating incrementally without detonating your release calendar.

---

### This Week's Big Story

**Migrate ViewController Navigation to SwiftUI NavigationStack**

Short story: a mature app’s navigation often becomes a web of push/pop calls, segues, and ad‑hoc hacks to support deep links, state restoration, and modal coordination. Moving to a value‑typed, state‑driven NavigationStack forces a single source of truth and surfaces routing decisions that are currently buried in view controllers. The hard part isn’t the API — it’s designing an adapter and rollout plan that doesn’t break payments, auth, or user expectations.

### Swift Snippet of the Week

```swift
import SwiftUI
import Observation

// Simple domain model
struct Item: Identifiable, Hashable {
    let id: UUID
    var title: String
}

// Observable app-level model using Swift Observation
@Observable
final class AppModel {
    var items: [Item] = [
        Item(id: .init(), title: "First"),
        Item(id: .init(), title: "Second"),
        Item(id: .init(), title: "Third")
    ]
    var isFlagged: Bool = false
}

// Navigation destinations for NavigationStack
enum Destination: Hashable {
    case list
    case detail(Item.ID)
}
// ... (truncated for newsletter)
```

This pattern encodes the operational decision to make routing a value‑typed, observable part of your app model so navigation becomes testable, auditable, and decoupled from fragile view controller side effects.
